### Outcome Notes

- Taxat now has an idempotent, resumable HMRC credential-export flow that:
  - adopts the existing sandbox application record
  - reads non-secret identifiers safely
  - blocks on a manual checkpoint before any one-time secret reveal unless the run explicitly allows reveal
  - writes revealed secret material straight into the governed vault writer
  - persists only sanitized metadata, lineage, vault refs, and attestation refs to repo-tracked outputs
- The new inventory and vault-binding records now attach:
  - callback profile refs
  - scope-set truth
  - fraud-header profile refs
  - token-binding profile refs
  - exact source references for each governed dimension
- The manual-checkpoint branch now emits schema-valid placeholder lineage and binding artifacts instead of leaving the export half-modeled.
- The viewer now exposes a calm security-ledger surface rather than dashboard-style output. It supports:
  - application selection
  - binding inspection
  - secret-version lineage inspection
  - evidence and attestation drawer inspection
  - safe copy for aliases, fingerprints, vault refs, and attestation refs only
- Repo-tracked outputs remain sanitized. Raw HMRC `client_id` and `client_secret` values are excluded from:
  - JSON artifacts
  - checked-in templates
  - viewer fixtures
  - test assertions and snapshots
- The remaining explicit gap is intentional and machine-registered: runtime token-exchange proof for the exported secret lineage remains deferred to later authority-sandbox execution cards.

Mandatory edge cases:
