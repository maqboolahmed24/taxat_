- `file_type`
- `sha256`
- `size_bytes`
- `authority_level` (`core_algorithm`, `canonical_contract`, `specialized_contract`, `enforcement`, `support_coherence`, `historical_closure`, `prompt_scaffold`, `noncanonical_residue`)

- `related_schema_files[]`
- `related_sample_files[]`

1. Recursively walk the live workspace and inventory every file under `Algorithm/` and `PROMPT/`.
2. Reuse the canonical-vs-residue split from task 001, but recompute it from the live tree instead of trusting prior output blindly.
3. For markdown files, capture title plus heading inventory. For schemas, capture `$id`, title, and sample-pair candidates.
4. Build a file-role taxonomy that can classify:
   - core algorithm and control-flow sources,
   - object/state/formula/protocol sources,
   - specialized UX/governance/portal/workspace contracts,
   - schema and sample artifacts,
   - validator and forensic guard code,
   - support/coherence documents,
   - historical closure documents,
   - prompt scaffolding,
   - archive residue.
5. Inventory schema/sample relationships. Where a sample is absent, emit an explicit note instead of leaving the relationship null-by-accident.
6. Detect orphaned or weakly classified files, including:
   - files not reachable from README inventory or support docs,
   - schemas with no obvious prose owner,
   - samples with no schema,
   - prompt scaffold files with no checklist/card relationship,
   - compiled/cache artifacts if present.
7. Build a heading-level JSONL inventory for markdown so downstream tasks can perform targeted extraction without re-parsing the full corpus.
8. Produce a topology overview that groups the corpus by authority layer and domain layer rather than just alphabetically.
9. Add validation logic that fails on duplicate paths, unstable sort order, missing checksums, or unclassified canonical files.

- Close the “README is only a partial inventory” gap by producing a full machine-readable file inventory.
- Close the “schema/sample ownership is implicit” gap by making every schema/sample relationship explicit.
- Close the “headings are trapped in prose” gap by exporting a searchable heading inventory.
- Close the “orphaned files are invisible” gap by surfacing unclassified or weakly anchored files in a dedicated gap register.
- Close the “prompt scaffold and algorithm corpus are mixed conceptually” gap by classifying them as separate but related artifact families.

Optional browser-viewable artifact:
- You MAY generate a local static inventory explorer if it materially improves inspection speed.
- If you do, keep it secondary to the canonical JSON/CSV outputs.
- Validate it with Playwright using semantic locators, keyboard navigation, heading/landmark checks, and trace capture.
- Keep the visual treatment premium-minimal and evidence-first: no decorative dashboard chrome, no noisy color gradients, no card clutter.

Mandatory edge cases:
- Preserve `schemas/` sample pairing even when naming is irregular.
- Distinguish true archive residue from Python cache or other workspace-local byproducts.
- The prompt scaffold may contain local absolute paths; treat them as metadata strings, not filesystem ground truth.
- If a file belongs to multiple logical domains, preserve a primary domain plus secondary tags instead of flattening nuance away.

Validation rules:
- Every canonical file must have exactly one primary authority-level classification.
- Every canonical markdown file must have a heading inventory row.
- Every schema file must either link to at least one sample or explicitly declare `no_sample_discovered`.
- No canonical file may remain unclassified.
- Output ordering must be deterministic across reruns.

Definition of done:
- A later agent can query the inventory to answer “what files exist, why they matter, who enforces them, and what domain they belong to”
  without rescanning the filesystem manually.
- The inventory clearly separates canonical corpus sources from prompt scaffolding and packaging residue.
