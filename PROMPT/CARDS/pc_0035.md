### Execution Notes

- `../shared_operating_contract_0030_to_0037.md` is still absent, so the implementation grounded directly in the named algorithm contracts, the environment/profile outputs from `pc_0031`, the provisioning workspace from `pc_0032`, the secret policy from `pc_0033`, the Developer Hub workspace record from `pc_0034`, and current official HMRC guidance.
- Revalidated the HMRC sandbox bootstrap and minimum-functionality guidance on `2026-04-18` before writing the baseline:
  - `Getting started` still says developers create a sandbox application, choose APIs, get credentials, then test.
  - `Testing in the sandbox` still says applications must be subscribed to APIs before they can access them and sandbox credentials are retrieved afterwards.
  - `How to integrate with HMRC APIs` still distinguishes the in-year required set from the end-of-year set.
  - `Application naming guidelines` still allow free-form sandbox naming while warning that production naming will be checked later.
  - `Development practices` still say one production application is normally sufficient and traffic isolation should come from OAuth tokens rather than per-customer app registrations.
- Normalized one machine-readable baseline in `data/provisioning/hmrc_required_api_set_baseline.json`:
  - required now: `Business Details (MTD)` `2.0`, `Obligations (MTD)` `3.0`, `Individual Calculations (MTD)` `8.0`, `Self Employment Business (MTD)` `5.0`, `Property Business (MTD)` `6.0`
  - likely required later: `Business Source Adjustable Summary (MTD)` `7.0`, `Individual Losses (MTD)` `7.0`
- Implemented two idempotent HMRC provider flows:
  - `register_sandbox_application.ts` adopts or creates the canonical sandbox app and persists a sanitized application record
  - `subscribe_required_mtd_apis.ts` reconciles the required-now subscriptions, records later-scope APIs as deferred, and updates the application record with the subscription matrix ref
- Extended the shared fixture portal instead of creating a second synthetic surface so both `pc_0034` and `pc_0035` run against the same HMRC-style Developer Hub model.
- Closed the card’s key gaps explicitly:
  - one canonical sandbox application record now exists
  - one required-versus-later API decision set now exists
