Required design decisions you must make explicit:
- Define the semantic boundary between IDs, refs, hashes, route tokens, storage refs, download refs, preview targets, and signed-delivery handles.

- Define whether locators are opaque strings, structured objects, or branded strings per family, and where human readability is allowed versus forbidden.
- Define how `delivery_binding_hash` and cache partition identity relate to preview/download affordances without leaking raw storage paths.
- Define how current-versus-history artifact selection influences default preview/download targets and what must never be inferred from raw object-store location.
- Define how locators survive moves, copy-on-publish, customer-safe derivatives, and provenance graph traversal.
- Define explicit forbidden content in locators: secrets, raw bearer tokens, direct signed URLs, mutable breadcrumb labels, and unsafe cross-tenant hints.

- Close the gap where raw object-store paths or signed URLs could creep into durable contracts and UI state.
- Close the gap where current/historical artifact meaning can drift because preview/download defaults are not tied to explicit target families.
- Close the gap where provenance and evidence tooling cannot reliably distinguish content location from object identity.
